## Conclusões gerais (Kimi)


1. **Leitura de faces e análise Z (Logs 001–014, 025–047)**
   - A leitura de cor via `feature.GetStyle` / `face.Style.Diffuse*` e a segmentação por proximidade/Z funcionam.
   - `Face.GetRange` requer `ParameterModifier` by-ref + `InvariantCulture`; sem isso dá `DISP_E_TYPEMISMATCH`.

2. **Inter-Part Copy (Logs 015–024)**
   - `CopySurfaces.Add` falha sistematicamente com `E_FAIL` quando chamado fora do modo de edição in-place real.
   - `Occurrence.Activate = true` NÃO entra em edição in-place; é apenas flag de ativação da ocorrência.
   - `InterpartConstructions.Add/Add2` também falhou com `E_FAIL`/`DISP_E_TYPEMISMATCH` para os tipos testados.
   - **Bloqueio conceitual:** a API COM pública do Solid Edge 2023 não expõe um método programático para iniciar edição in-place.
   - **Próxima hipótese:** usar `Face.GetReferenceKey` + `Occurrence.CreateTopologyReference` como substituto.

3. **Modelagem de bloco (Logs 008–011, 033–047)**
   - Sketch + extrusão (`ProfileSets.Add → Profiles.Add → Lines2d.AddBy2Points → Profile.End(1) → Models.AddFiniteExtrudedProtrusion`) é o caminho validado.
   - `AddBoxByTwoPoints` e primitivos diretos repetidamente falham com `DISP_E_TYPEMISMATCH`/`DISP_E_PARAMNOTOPTIONAL` no late binding.
   - Furos simples (`igRegularHole=33`) via `Holes.AddSync` funcionam no síncrono (Log 045: 3/3 eletrodos criados).

4. **Furos roscados M6 (Logs 046–047)**
   - `HoleDataCollection.Add(37, ...)` (`igTappedHole`) cria o `HoleData`, mas `Holes.AddSync` retorna `E_FAIL`.
   - Após o `E_FAIL` do M6, o proxy COM é corrompido (`0x80010114`) e aborta o eletrodo.
   - **Fallback adotado:** usar furo simples Ø5 (broca de rosca) e deixar a rosca para operação manual.
   - **Caminho futuro:** testar `Model.Threads.Add(HoleData, nCylinders, CylinderArray, CylinderEndArray, ...)` para renderizar rosca separadamente.

5. **Unidades e marshaling**
   - API de geometria usa metros e radianos.
   - Arrays de COM objetos (`Face[]`, `Profile[]`) precisam ser tipados para `SAFEARRAY(IDispatch)`.
   - Parâmetros `[out]` precisam de `ParameterModifier` by-ref e `CultureInfo.InvariantCulture`.

6. **Posicionamento de eletrodos**
   - `Occurrence.GetTransform` retorna metros; `PutOrigin` espera metros.
   - `ElectrodeBuilder.CreateInContextPart` divide a origem por `1000.0` (bug de unidade) — deve ser corrigido.
   - `CreateInContextPart` permanece bloqueado pelo Inter-Part Copy; o fluxo funcional é standalone (`Documents.Add → BlankModeler → SaveAs → AddByFilename → PutOrigin`).
